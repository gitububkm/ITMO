5)TL;DR
Нужно создать простой фронтенд на React для уже существующего API новостей (лабораторные работы 2–5), используя Vite.
Приложение должно уметь:
показывать список новостей;
отображать страницу конкретной новости с комментариями;
выполнять запросы к API через fetch или axios;
работать с авторизацией (отправка токена, скрытие/отображение действий в зависимости от роли пользователя).
Создание проекта — обязательно через Vite:
Документация: https://vite.dev/guide/
Команда инициализации: npm create vite@latest
Необходимо создать новый репозиторий в организации и приложить ссылку на него.
Сценарий использования
Запустить локально фронтенд-приложение.
Подключить его к API, реализованному в лабораторных работах 2–5 (новости, права, комментарии, авторизация).
Через интерфейс приложения: увидеть список новостей, открыть страницу новости и увидеть её содержимое и комментарии, авторизоваться, создать комментарий к новости от имени авторизованного пользователя, отредактировать или удалить новость/комментарий
Убедиться, что запросы к API выполняются через fetch или axios, ответы корректно отображаются в интерфейсе.
Зафиксировать в README.md примеры запросов (какие страницы есть, какие ручки дергаются).
Техническое задание
Функциональные требования
Структура приложения. Приложение должно содержать как минимум три экрана:
Главная страница (список новостей). Отображает список новостей с заголовком, датой, именем автора, по желанию обложкой. При переходе по элементу списка открывается страница конкретной новости.
Страница одной новости. Отображает заголовок, дату публикации, автора, контент, список комментариев. На этой странице должна быть форма добавления нового комментария: поле ввода текста и кнопка отправки, выполняющая запрос к API с привязкой к текущей новости и авторизованному пользователю.
Страница авторизации. Простая форма с полями email/логин и пароль. При отправке данных вызывается эндпоинт авторизации из ЛР 3, полученный access-токен сохраняется (например, в localStorage). При наличии токена в интерфейсе следует отображать информацию о текущем пользователе и кнопку выхода, а также добавлять заголовок Authorization в запросы, где это требуется.
Работа с ролями и правами. Необходимо использовать ролевую модель и флаг «верифицирован ли как автор» из ЛР 3.
Неавторизованный пользователь может просматривать список новостей и комментарии, но не может создавать новости и не может редактировать или удалять новости и комментарии.
Авторизованный пользователь без верификации как автор может оставлять комментарии, но не может создавать новости.
Верифицированный автор может создавать новости
В интерфейсе для него должна быть доступна возможность создать новость (например, кнопка «Создать новость» на главной странице или отдельный экран создания)
Форма создания новости должна содержать заголовок, контент
Пользователь с ролью администратор может выполнять все операции: редактировать и удалять любые новости, удалять любые комментарии
Достаточно базовых кнопок и проверок видимости элементов интерфейса в зависимости от роли, которую возвращает API.
Запросы к API
Все запросы к бекенду должны выполняться через fetch или через axios
